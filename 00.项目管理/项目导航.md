# 项目导航

本页用于判断任务入口、文件归属和文档关系。

项目操作规则见[交易项目规则](../AGENTS.md)，通用规则见[上级知识库规则](../../AGENTS.md)。当前 SOP 与模板以[当前生效版本](当前生效版本.md)为准；文档状态、更新方式和经验升级条件见[属性与状态规范](属性与状态规范.md)。

## 快速入口

| 当前任务 | 首先读取 | 后续操作 |
|---|---|---|
| 交易分析 | [当前生效版本](当前生效版本.md) | 读取指定 SOP，按 SOP 完成分析 |
| 创建交易计划 | [当前生效版本](当前生效版本.md) | 读取计划模板，写入 `03.交易记录/01.交易计划` |
| 记录实际交易 | 对应交易计划和日志模板 | 写入 `03.交易记录/02.交易日志` |
| 单笔复盘 | 对应计划、日志和复盘模板 | 写入 `04.交易复盘/00.单笔复盘` |
| 日、周、月复盘 | 相关交易与下级复盘 | 写入对应复盘目录 |
| 提炼经验 | 相关记录和复盘 | 读取[属性与状态规范](属性与状态规范.md)，先进入待验证经验 |
| 验证或升级经验 | 经验卡及其证据 | 按[属性与状态规范](属性与状态规范.md)处理 |
| 研究新策略或工具 | 相关参考资料 | 写入 `02.策略研究`，不直接改变 SOP |
| 查询概念、公式或接口 | `06.交易知识与参考` | 读取对应分类文档 |
| 提出 SOP 修改 | 已验证经验和相关证据 | 在 `02.策略研究` 创建修改提案 |
| 查找当前规则 | [当前生效版本](当前生效版本.md) | 不使用归档版本替代当前版本 |

## 目录职责

| 目录 | 负责内容 | 不应存放或执行 |
|---|---|---|
| `00.项目管理` | 项目导航、当前生效版本、属性与状态规范 | 具体交易规则和原始交易记录 |
| `01.交易SOP` | 当前及待确认的正式 SOP | 交易日志、复盘、一般知识说明 |
| `02.策略研究` | 策略假设、回测、观察、工具改进、SOP 修改提案 | 未经确认的当前交易规则 |
| `03.交易记录/01.交易计划` | 入场前制定的交易计划 | 根据结果补写或改写的计划 |
| `03.交易记录/02.交易日志` | 入场、出场、仓位、费用、滑点和执行偏差等事实 | 脱离事实证据的通用经验 |
| `04.交易复盘/00.单笔复盘` | 单笔交易的计划与执行对照 | 对原始计划和日志的修改 |
| `04.交易复盘/01.日复盘` | 当日交易与纪律执行总结 | 未经验证的长期规则 |
| `04.交易复盘/02.周复盘` | 一周内重复行为、策略表现和风险问题 | 当前 SOP |
| `04.交易复盘/03.月复盘` | 长期稳定性、适用环境和风险变化 | 原始成交事实的替代记录 |
| `05.交易经验库/01.待验证经验` | 有证据来源但尚未充分验证的经验假设 | 已确认生效的 SOP 规则 |
| `05.交易经验库/02.已验证经验` | 经过验证并注明适用边界的经验 | 自动覆盖当前 SOP 的规则 |
| `05.交易经验库/03.失败模式` | 已确认且可重复识别的负面模式 | 尚未确认的失败假设 |
| `06.交易知识与参考` | 市场机制、技术分析、合约风险、API 和工具资料 | 当前交易指令 |
| `90.模板` | 正式笔记的属性、字段和结构 | SOP 中不存在的交易规则 |
| `99.归档` | 已失效版本和历史资料 | 当前有效规则 |

## 内容归属判断

整理内容时，先判断它在回答什么问题：

- “现在应该按照什么规则交易？” → `01.交易SOP`
- “这个想法是否有效？” → `02.策略研究`
- “交易前准备怎么做？” → `03.交易记录/01.交易计划`
- “实际发生了什么？” → `03.交易记录/02.交易日志`
- “为什么出现这个结果或偏差？” → `04.交易复盘`
- “这个结论是否可以重复使用？” → `05.交易经验库`
- “这个概念、公式或工具怎么工作？” → `06.交易知识与参考`
- “这类笔记应该填写哪些字段？” → `90.模板`
- “已经失效但需要保留什么？” → `99.归档`

一篇内容同时回答多个问题时，应拆分成不同文档并通过链接关联，避免复制同一规则。

## 文档关联

- 交易日志应链接对应交易计划。
- 单笔复盘应链接对应交易计划和交易日志。
- 日复盘应链接相关交易或单笔复盘。
- 周复盘应链接相关日复盘和代表性交易。
- 月复盘应链接相关周复盘及核心证据。
- 经验卡应链接支持或反驳它的记录与复盘。
- SOP 修改提案应链接已验证经验及相关证据。
- 新版 SOP 应说明对应的修改提案。

不得使用“根据以前交易”等无法追溯的笼统描述代替证据链接。

## 项目工作流

### 交易证据链

交易计划  
→ 交易日志  
→ 单笔复盘  
→ 日/周/月复盘  
→ 待验证经验  
→ 已验证经验  
→ SOP 修改提案  
→ 用户确认  
→ 新版 SOP  
→ 更新[当前生效版本](当前生效版本.md)

### 外部研究链

外部资料或工具想法  
→ 策略研究  
→ 回测、模拟或观察  
→ 待验证经验  
→ 已验证经验  
→ SOP 修改提案

未经验证的研究结论不得直接进入当前 SOP；经验升级和文档更新方式按照[属性与状态规范](属性与状态规范.md)执行。
